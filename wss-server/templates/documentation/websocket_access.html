Websocket

Haozheng Li
Create Date : 4 Mar, 2023
New

WSS platform supports access to devices with Websocket protocol. This article introduces the access process of using Websocket connection communication.

Restrictions

  • Only directly connected devices support communication using the Websocket protocol. Gateway devices and gateway sub-devices do not support communication using the Websocket protocol.
  • Websocket protocol access only supports Websocket request paths with TLS encryption. (e.g: wss://)
  • The size of data transmitted by the data uplink interface is limited to 128 KB.

Access process

1. Connection verification

Put the api-key parameter that can verify the device into the websocket request header, and make a websocket connection with the WSS server

  • URL: wss://wssweb.net/ws/device/<api-key>
  • Replace <api-key> to your device api key
2. message body format
message_type should be one of: <text, performance, event>
message json data

Example

Here is a Python client instance with websocket-client. Please install websocket-client first

import websocket
import json
def on_message(wsapp, message):
    print(message)

wsapp = websocket.WebSocketApp("wss://wssweb.net/ws/device/<api-key>", on_message=on_message)
test_message = {'message_type': 'text', 'message': 'Hello WSS'}
wsapp.send(json.dumps(test_message))
wsapp.run_forever()

For more example of using an Websocket client to access the WSS, see wss-client.
